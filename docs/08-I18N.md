# 08 - Internationalization (English / Spanish)

## 1. Strategy

- **Primary language:** English (en)
- **Secondary language:** Spanish (es)
- **Library:** next-intl (best Next.js App Router i18n solution)
- **Switching:** User can switch language anytime from settings or home screen
- **Persistence:** Language preference saved to profile (or localStorage for guests)
- **URL structure:** `/en/galaxy/...` and `/es/galaxy/...`
- **Math symbols:** Universal (no translation needed for `+`, `-`, `×`, `÷`, `=`)

## 2. Implementation Setup

### next-intl Configuration

```typescript
// src/i18n/config.ts
export const locales = ['en', 'es'] as const;
export type Locale = (typeof locales)[number];
export const defaultLocale: Locale = 'en';
```

```typescript
// src/i18n/request.ts
import { getRequestConfig } from 'next-intl/server';

export default getRequestConfig(async ({ locale }) => ({
  messages: (await import(`./messages/${locale}.json`)).default,
}));
```

```typescript
// src/middleware.ts
import createMiddleware from 'next-intl/middleware';
import { locales, defaultLocale } from './i18n/config';

export default createMiddleware({
  locales,
  defaultLocale,
  localePrefix: 'always',
});

export const config = {
  matcher: ['/((?!api|_next|.*\\..*).*)'],
};
```

### Usage in Components

```tsx
import { useTranslations } from 'next-intl';

export function GameHeader() {
  const t = useTranslations('game');

  return (
    <header>
      <h1>{t('title')}</h1>
      <p>{t('level', { number: 5 })}</p>
      <p>{t('streak', { count: 7 })}</p>
    </header>
  );
}
```

## 4. Dynamic Content Translation

Some content is generated dynamically and needs translation handling:

### Math Operation Symbols (Universal)

```typescript
const OPERATION_SYMBOLS: Record<Operation, string> = {
  addition: '+',
  subtraction: '−',       // minus sign (U+2212), not hyphen
  multiplication: '×',    // multiplication sign (U+00D7)
  division: '÷',          // division sign (U+00F7)
};
```

## 5. Language Switcher Component

```tsx
'use client';
import { useLocale } from 'next-intl';
import { useRouter, usePathname } from 'next/navigation';
import { Locale, locales } from '@/i18n/config';

const LANGUAGE_LABELS: Record<Locale, { flag: string; label: string }> = {
  en: { flag: '🇺🇸', label: 'English' },
  es: { flag: '🇪🇸', label: 'Español' },
};

export function LanguageSwitcher() {
  const locale = useLocale();
  const router = useRouter();
  const pathname = usePathname();

  function switchLocale(newLocale: Locale) {
    const newPath = pathname.replace(`/${locale}`, `/${newLocale}`);
    router.push(newPath);
  }

  return (
    <div className="flex gap-2">
      {locales.map((loc) => (
        <button
          key={loc}
          onClick={() => switchLocale(loc)}
          className={loc === locale ? 'font-bold' : 'opacity-60'}
        >
          {LANGUAGE_LABELS[loc].flag} {LANGUAGE_LABELS[loc].label}
        </button>
      ))}
    </div>
  );
}
```

## 6. RTL Consideration

Neither English nor Spanish requires RTL support. If Arabic or Hebrew are added in the future, the layout system should be updated with `dir="rtl"` and logical CSS properties (`margin-inline-start` instead of `margin-left`).
